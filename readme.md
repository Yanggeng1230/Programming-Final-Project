## 程式結構
### 主程式 `main.py`
遊戲程式運行的起點，預期會產生一個遊戲畫面的視窗，所有遊戲流程都在該視窗內執行。

### 角色行為 `character.py`
#### 教授 professor
屬性有：
- 名字(name)
- 領域(field)
- 教學科目(subject) 
- 與主角的親密度(affinity)
- 位置座標(position)

玩家可以透過與教授聊天的方式，增加親密度並強化對應領域的能力。
教授資料都記錄在 `professor.json` 裡，預計會隨機出現在地圖中，以下是教授資料的示例：
```json
{
    "professors": [
        {
            "name": "楊青育",
            "field": "Analysis",
            "subjects": ["程式設計", "電子計算機概論", "微積分"]
        },
        // 其他教授繼續往下放
    ]
}
```

#### 數學家怨靈 boss
屬性有：
- 名字(name)
- 領域(field)
- 血量(health)
- 位置座標(position)

玩家可以用各領域的技能攻擊他，能力值越高則傷害越高，如果使用的技能正好是怨靈的領域，傷害會再進一步提升。
怨靈資料都記錄在 `boss.json` 裡，預計會隨機挑一個出現在地圖中，以下是怨靈資料的示例：
```json
{
    "boss": {
        "name": "伽羅瓦",
        "field": "Algebra",
        "health": 100
    },
    // 其他怨靈繼續往下放
}
```

#### 玩家 player
屬性有：
- 名字(name)
- 性別(sex)
- 所在地圖名稱(map)
- 位置座標(position)
- 血量(health)
- 各領域能力值(analysis, algebra, geometry)

方法有：
- 朝某個方向移動 `move(direction)`
- 撿拾物品 `get_item()`
- 與教授聊天 `talk(prof: professor)`
- 與怨靈戰鬥 `battle(enemy: boss)`

### 地圖 maps/Gongguan.json
預計讓主程式讀取的地圖，記錄寬度、高度、各地點名稱及牆壁位置(玩家只能走沒有牆壁阻擋的路徑)
以下是地圖檔的示例：
```json
{
  "width": 3, //寬度
  "height": 3, //高度
  "locations": [ //各地點名稱
    {"pos": [0, 0], "name": "校門口"},
    {"pos": [0, 2], "name": "圖書館"},
    {"pos": [2, 2], "name": "學生餐廳"}
  ],
  "walls": [ //牆壁位置
    {"from": [0, 0], "to": [1, 0]},
    {"from": [1, 1], "to": [1, 2]}
  ]
}
```

## 其他有用的提示
### 什麼是 json
一種方便的資料格式，簡單理解成 python 裡的字典就好
### 關於地圖
`地圖.pptx` 是預先編輯地圖的用的，地圖定稿後可以使用 `範例 by AI` 裡的`地圖設計器3.py` 來產生地圖檔
### 關於NPC
1. 教授：去系網找有印象的教授，按照領域和教學科目填進去就好
2. 怨靈：去網路上翻看看有名的數學家，一樣按照領域填進去就好，或許名氣和血量有正相關(?)
